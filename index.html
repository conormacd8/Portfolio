What the project aimed to solve

While working on this project, my main goal was to utilize the amazing tools of TailwindCSS and create something that I would be proud to show off to potential future employers. During the development process I learned alot about TailwindCSS and how to use the Git Version Control System to upload my work to github. I came across a few issues along the way whilst learning and will briefly discuss what those issus were and how they were overcome to make the project better in the next section.

Problems I encountered

Quite early on in the process it took me a while to get a proper grasp of the new way to write CSS with TailwindCSS. Some of the things that I found difficult with TailwindCSS was the naming of certain CSS properties such as 'rounded' for 'border-radius', 'tracking' for 'letter-spacing', and 'leading' for 'line-height'. Although I quickly overcame the issues I had with TailwindCSS's property names, what took me alot more time to overcome was trying to use certain Javascript features with TailwindCSS. Initially, I struggled to create simple CSS components such as an animated mobile navigation and pop-up modal for the modal that you are viewing right now, but as discussed in the next section those issues were overcome succesfully.

How they made the project better

By the end of the project, I overcame the obstacles I previously identified by reading through the official documentation and finding people who went through similar problems as me online and learnt from their mistakes. I learnt how to utilize the CSS framework to help improve my workflow and intend to implement the framework on future projects where I see fit which will allow me to invest more time in the more complex and thought provoking portion of my projects, the Javascript.

Javascript card pairing game

Javascript

Tailwind

HTML5

Match the card guessing game

What the project aimed to solve

My intention behind building this project was primarily to improve my skills with DOM manipulation and traversal. Although small, there was a few things I had to consider when building this project out such as how to manipulate the DOM in the least performance hindering way, keep track of which cards have been selected and ensuring that those already clicked cards can not be clicked again, keeping track of the correct cards selected and much more.

Problems I encountered

Whilst working on the project, I encountered a few problems. One such problem that stands out was how I was going to identify which card/character had been selected and whether or not its pair was also selected or not. I initially thought I could just use class names and assign the same class name to each card pair but after some research I learnt about the 'data' attribute which I found really interesting.

How they made the project better

I was able to use the data attribute to seperate my CSS styling in my classes and the custom data attributes that I needed to select which helped make development easier to keep track of as TailwindCSS can become quite cluttered with all of its class names.

What the project aimed to solve

As part of a Scrimba.com project, I built this Personal Dashboard Chrome Extension. The goal of this project was to become more comfortable interacting with and extracting data from Web API's. Some of the Web API's utilized for this project include 'CoinGecko', 'OpenWeatherMap', and 'Unsplash'.

Problems I encountered

One of the problems that I encountered during the development of this project were of course learning how to utilize Web API's as it was a first for me. How they made the project better

Utilizing these different Web API's was a really enjoyable experience and looking at it now. It would have been extremely difficult to get this dynamic data on my own without them and as a result being able to implement these API's added alot of value to the project.
